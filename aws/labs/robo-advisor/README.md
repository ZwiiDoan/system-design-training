# Robo-Advisor Infrastructure

This Terraform project deploys a scalable and resilient "User Onboarding" service on AWS. It is designed to serve as a foundational example of a production-ready infrastructure setup, featuring a VPC, an Application Load Balancer (ALB) distributing traffic across an Auto Scaling Group (ASG) of EC2 instances.

## Architecture Overview

The architecture consists of the following key components:

- **VPC**: A logically isolated section of the AWS Cloud where you can launch AWS resources in a virtual network that you define.
- **Application Load Balancer (ALB)**: Serves as the single point of contact for clients, distributing incoming traffic across multiple targets (EC2 instances) in different Availability Zones. It is configured with a listener on port 80 and a target group for health checks.
- **Auto Scaling Group (ASG)**: Manages a fleet of EC2 instances, automatically scaling the number of instances up or down based on CPU utilization. This ensures that the application has the capacity to handle the current load while minimizing costs.
- **Launch Template**: Defines the configuration of the EC2 instances launched by the ASG, including the AMI, instance type, user data, and security groups.
- **Security Groups**: Act as a virtual firewall, controlling the traffic to and from the ALB and EC2 instances.
